It does not copy third-party formulas, UI, language, or proprietary behavior. Physical validation is required before any sensor, BLE, HealthKit, or haptic feature can be described as device-validated.

## Device-First Update

This pass added a device-first source resolver and local-day aggregator. Reliable wearable BLE direct measurements now outrank Apple Health where the decoded signal is safe to use. Apple Health remains the fallback for steps, sleep, workouts, HRV, RHR, respiratory rate, oxygen saturation, VO2, temperature, active energy, and distance when wearable data is missing or not yet validated. Raw IMU does not create steps. BPM-only data does not create true HRV. Raw PPG does not create production SpO2.
